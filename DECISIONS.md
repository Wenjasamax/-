# DECISIONS

1. Выбран стек `PHP 8.2 + MySQL + Nginx` в Docker Compose, чтобы запуск был одинаковым у проверяющего и разработчика.
2. Авторизация упрощена до выбора пользователя из сидов на странице входа, как разрешено условием задания.
3. Статусная модель заявок реализована строго по ТЗ: `new`, `assigned`, `in_progress`, `done`, `canceled`.
4. Защита от гонки для "Взять в работу" сделана на уровне БД через атомарный `UPDATE ... WHERE status = 'assigned'` и проверку `rowCount()`.
5. Для действий добавлен журнал `request_events` (audit log), чтобы упростить отладку и демонстрацию жизненного цикла заявки.
6. Миграции и сиды запускаются автоматически при старте контейнера `app`, чтобы после `docker compose up` система сразу была готова к проверке.
7. Автотесты вынесены в unit-уровень на `phpunit` с in-memory SQLite, что ускоряет проверки бизнес-логики без поднятия MySQL.
